# cold-b2b-email-ru

Codex skill для холодных B2B-писем на русском языке.

Skill помогает писать первое холодное письмо и follow-up цепочку 3-5 касаний так, чтобы письмо звучало как сообщение от сильного B2B-продавца: лично, конкретно, без давления и без шаблонного AI-языка.

## Для Чего

Используйте skill, когда нужно:

- написать первое холодное B2B-письмо
- собрать цепочку follow-up на 3-5 касаний
- получить ответ
- назначить звонок
- добиться пересылки HR / CEO / ответственному
- получить разрешение прислать сценарий пилота
- пригласить на презентацию продукта
- отработать молчание после первого касания

## Как Работает

При запуске skill сначала спрашивает:

1. Какая цель письма?
2. Какой ожидаемый результат?

Затем уточняет:

- кому пишем
- нужна ли цепочка 3-5 писем
- нужны ли версии под разные боли, сегменты или роли
- вставлять ли ссылку и какую
- какие proof points можно использовать
- есть ли запретные слова или claims
- какой следующий шаг в продажах

Если в проекте есть контекст, skill использует его: `AGENTS.md`, `.agents/product-marketing-context.md`, `.claude/product-marketing-context.md`, sales notes или расшифровки звонков.

## Принципы

- Одно письмо, одна задача.
- Письмо звучит как человек, а не как шаблон.
- Первый экран письма про мир клиента, не про продукт.
- CTA легкий для ответа в одну строку.
- Proof points используются только после проверки, что их можно использовать.
- Follow-up добавляет новый смысл, а не пишет "просто напоминаю".
- Нельзя использовать fake `Re:` / `Fwd:`, ложную срочность, давление, завышенные promises и непроверенные claims.

## Нативность Русского Письма

- Не переводить английские sales-обороты дословно.
- Писать уважительно, но живо: чаще `вы`, без панибратства.
- Сначала показывать бизнес-симптом, потом боль или гипотезу.
- Не перегружать письмо тире, англицизмами и абстрактным HR-языком.
- CTA формулировать так, чтобы человеку было легко ответить или переслать письмо ответственному.

## Безопасность

Skill текстовый: внутри нет скриптов, токенов, сетевых вызовов или скрытой автоматизации.

Для чувствительных тем skill просит ограничения проекта перед написанием: медицинские claims, финансовые обещания, внутренние метрики, клиентские имена, юридические ограничения, запрещенные формулировки.

Для массового outbound пользователь отвечает за законность базы, отписку, sender reputation, bounce rate, spam complaints и региональные требования.
